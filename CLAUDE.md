# CLAUDE.md — AS ONE プロジェクト

## プロジェクト概要

**AS ONE** は、サッカークラブの選手・保護者・コーチ・経営陣の「関係性・評価・対話」を統合し、育成と組織運営を高度化する **SPA型Webアプリケーション**。

単なる業務効率化ツールではなく、以下を実現する基盤として設計されている：

- **チームビルディングOS**：チームを早期に統一期（高い一体感）へ導く
- **三者関係の安定化装置**：選手・保護者・コーチ間のズレを可視化・解消する
- **指導者成長支援基盤**：コーチの人間性・テイストを引き出す仕組み

### コア設計思想

| 思想 | 説明 |
|------|------|
| 仕組み × テイストの両立 | 仕組みは「下支え」、テイストが「価値を生む」 |
| 余白設計 | 完全最適化しない。あえてズレ・不完全を残し、人間の成長を促す |
| 想定外の創出 | ベースを整えることで想定外が発生し、それが創造性になる |

## 現在の実装状態

### フェーズ：Vanilla JS プロトタイプ

現在のコードベースは React 移行前のプロトタイプ。以下のファイル構成：

```
asone/
├── index.html    # UI + CSS + 構造（一体化）
├── app.js        # 状態管理・画面制御・ロジック
├── data.js       # モックデータ（選手・スタッフ・週報など）
└── style.css     # スタイル
```

### 実装済み機能
- ロール別ログイン（admin / coach / player / parent）
- ダッシュボード、選手情報管理、スタッフ管理
- 選手評価（心・頭・技・体）
- 週報管理、業務チェックリスト、お知らせ

## 目標アーキテクチャ（SPA版）

### システム構成

```
[React SPA（フロントエンド）]
        ↓ REST API / WebSocket
[NestJS Backend]
        ↓
[PostgreSQL DB]
        ↓
[AI API / 通知 / 外部連携]
```

### 3レイヤー設計

| レイヤー | 役割 |
|---------|------|
| ① AIコミュニケーション | 選手・保護者とのAIチャット。コーチの分身として機能 |
| ② データ統合・分析 | チャット蓄積・アウトプット可視化・分析レポート生成 |
| ③ 人間コーチ連携 | AI→人間へのフィードバック、担当コーチとのすり合わせ、最終判断 |

## 権限設計

### RBAC（ロールベース）

| ロール | 説明 |
|--------|------|
| `admin` | 経営陣。全機能にアクセス可 |
| `management` | 管理職。クラブ運営管理 |
| `coach` | コーチ。担当選手のみ編集可 |
| `player` | 選手。自分のデータのみ編集可 |
| `parent` | 保護者。自分の子のデータのみ閲覧可 |

### ABAC（条件付きアクセス）
- coach → 担当選手のみ編集可
- parent → 自分の子のみ閲覧可
- player → 自分のみ編集可

## AIの役割（Claude Code）

このプロジェクトにおいて Claude Code は以下の役割を担う：

### 開発支援
- Vanilla JS プロトタイプから React SPA への移行作業
- コンポーネント設計・API設計のレビューと実装
- データベーススキーマ設計のサポート

### 設計方針の遵守
- **余白設計**を尊重する：過度な自動化・完全最適化を提案しない
- **テイスト（個性）** を損なわない実装を優先する
- 三者関係（選手・保護者・コーチ）の文脈を常に意識した設計を行う

### コーディングルール
- TypeScript を使用する場合は型を明示的に定義する
- コンポーネントは単一責任の原則に従い、小さく保つ
- AIチャット機能はプロンプト設計をコーチの「テイスト」に合わせて調整可能な構造にする
- セキュリティ：RBAC/ABACの権限チェックはバックエンドで必ず行う（フロントのみに依存しない）

### 禁止事項
- 権限チェックをフロントエンドのみで完結させること
- ユーザーの個人データ（選手・保護者情報）をログに出力すること
- AIが最終判断・個別対応を行うこと（必ず人間コーチにエスカレーションする設計にする）
